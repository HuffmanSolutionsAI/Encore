# Encore infrastructure (Terraform)

The AWS infrastructure for Encore. The build spec defaults to Supabase; this
project instead runs on AWS at Jake's direction — see the root `README.md`.

## What this provisions

| Area | Resource |
|------|----------|
| Network | VPC, public + private subnets across 2 AZs, NAT gateway |
| Database | RDS for PostgreSQL 16 (private subnets) |
| Auth | Cognito user pool + app client, Sign in with Apple identity provider |
| Compute | API Lambda (Node 20) in the VPC |
| API | API Gateway HTTP API with a Cognito JWT authorizer |
| Secrets | Secrets Manager — database credentials and third-party API keys |

## Files

```
versions.tf    Terraform + provider versions, shared data sources and locals
variables.tf   Input variables
network.tf     VPC, subnets, routing, NAT
rds.tf         PostgreSQL instance + its Secrets Manager secret
cognito.tf     User pool, app client, Apple identity provider
lambda.tf      API Lambda, IAM role, log group
apigw.tf       HTTP API, routes, JWT authorizer, stage
secrets.tf     Third-party API key secret
outputs.tf     API URL, Cognito ids, database endpoint
```

## Deploy

```sh
cd ../api && npm install && npm run build    # build the Lambda bundle first
cd ../infra
cp terraform.tfvars.example terraform.tfvars # then fill in values
terraform init
terraform apply
```

`terraform apply` fails if `../api/dist` does not exist — build the API first.

## Secrets

`terraform.tfvars` (gitignored) holds the Sign in with Apple credentials. The
RDS master password is generated by Terraform. Last.fm / MusicBrainz keys are
created as empty Secrets Manager values and set out-of-band:

```sh
aws secretsmanager put-secret-value \
  --secret-id encore-dev/thirdparty \
  --secret-string '{"LASTFM_API_KEY":"...","LASTFM_SHARED_SECRET":"...","MUSICBRAINZ_USER_AGENT":"Encore/1.0 ( contact@example.com )"}'
```

## Notes

- Remote state is not configured — add an S3 backend before collaborating.
- `db_multi_az` defaults to `false` for cost; set it `true` for production.
- The RDS instance is not publicly accessible; run migrations from inside the VPC.
